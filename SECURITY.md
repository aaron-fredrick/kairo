# Security Policy

## Supported Versions

Please see the following table for information regarding which versions of Kairo are currently being supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of Kairo very seriously. If you discover a security vulnerability within Kairo, please send an e-mail to the maintainers rather than creating a public issue.

1. **Do not open up a GitHub issue** if the bug is a security vulnerability in Kairo.
2. Email your findings to `security@kairo.internal` (placeholder email).
3. We will acknowledge receipt of your vulnerability report within 48 hours.
4. We will provide an estimated timeline for the fix and keep you updated.
5. Once the issue is resolved, we will publish a security advisory.

Thank you for helping keep Kairo and our users safe!
